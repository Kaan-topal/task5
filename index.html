DNS and Mail System Overview

1. DNS System

1.1. List of Top-Level Domains (TLDs)

Top-Level Domains (TLDs) are the highest level in the Domain Name System hierarchy and appear at the end of a domain name:

  • Generic TLDs (gTLDs): .com, .org, .net, .info, .edu, etc.
  • Country Code TLDs (ccTLDs): .uk, .de, .fr, .jp, etc.
  • New gTLDs: .xyz, .online, .app, etc.

1.2. Hierarchical Structure of the DNS

The Domain Name System (DNS) is arranged in a hierarchical structure:

  • Root Level: Represented by "." and managed by root servers.
  • Top-Level Domains (TLDs): For example, .com, .org, .uk.
  • Second-Level Domains: Under a TLD, e.g., example.com.
  • Subdomains: Under a second-level domain, e.g., www.example.com.

When a client requests a domain, DNS resolvers check their cache or query up the chain: local DNS → root servers → TLD servers → authoritative name servers.

DNS Delegation: Responsibility passes down each level of the hierarchy. The root delegates to TLDs, TLDs delegate to second-level domains, and so on.

1.3. DNS Records, DNS and Reverse DNS

DNS servers store various DNS records:

  • A/AAAA: Map a domain to an IPv4/IPv6 address.
  • CNAME: Alias one domain name to another.
  • MX: Specify mail servers handling email for a domain.
  • NS: Indicate authoritative name servers for a domain.
  • TXT: Arbitrary text data, often for security/verifications (e.g., SPF).

Reverse DNS: Maps an IP address back to a domain name using PTR records, commonly used to verify email server authenticity.

1.4. Authoritative vs. Non-Authoritative Answers

Authoritative Answers: Directly from the DNS server that holds the domain’s DNS data.

Non-Authoritative Answers: From a DNS resolver’s cache or another non-authoritative source, potentially out-of-date.

1.5. DNS Resolver, DNS Cache, and /etc/hosts

DNS Resolver: The component (often a local or ISP-provided server) that queries DNS records on behalf of clients.

DNS Cache: Stores DNS query results temporarily to speed up future lookups until the TTL expires.

/etc/hosts: A local file that can map hostnames to IP addresses before DNS queries are made.

2. Mail System

2.1. Protocols: POP3, IMAP, SMTP, Client-Server Communication

SMTP (Simple Mail Transfer Protocol): Used for sending emails from clients to servers and between mail servers.

POP3 (Post Office Protocol v3): Downloads messages from the server to the client’s machine, often removing them from the server.

IMAP (Internet Message Access Protocol): Synchronizes mail between client and server, supports multiple folders, and keeps mail on the server.

Basic commands:

  • SMTP: HELO, MAIL FROM:, RCPT TO:, DATA, QUIT
  • POP3: USER, PASS, LIST, RETR, DELE
  • IMAP: LOGIN, SELECT, FETCH, STORE, LOGOUT

2.2. NNTP Protocol

NNTP (Network News Transfer Protocol): Used for reading and posting articles to Usenet newsgroups. Less common now, but historically significant for distributed discussion boards.

2.3. E-mail Mailbox, Parameters, Aliases, Address

Mailbox: Stores a user’s incoming messages on the server.

Aliases: Alternative addresses that deliver mail to the same mailbox, e.g., info@example.com → john@example.com.

E-mail Address: Format is local-part@domain, e.g., john.doe@example.com.

2.4. E-mail Structure: Header, Separator

An email consists of:

  • Headers: Metadata like From:, To:, Subject:, Date:
  • Body: The main content. Headers and body are separated by a blank line.

2.5. Attachments, E-mail Coding, MIME

Attachments: Files sent with an email, encoded for safe transmission.

MIME (Multipurpose Internet Mail Extensions): Standard for sending non-ASCII text and binary data like images or PDFs. Defines Content-Type and Content-Transfer-Encoding.

E-mail Coding: Techniques like Base64 or Quoted-Printable ensure attachments and non-ASCII text can be transmitted over text-only protocols.

2.6. Mailing Lists, E-mail Gate

Mailing Lists: Allow group distribution of emails to multiple subscribers via a single address.

E-mail Gate (Gateway): Converts or routes emails between different systems or formats, bridging otherwise incompatible services.

2.7. E-mail Security: SSL, TLS, PGP, Signature

SSL/TLS: Encrypts email transmission (e.g., SMTPS, IMAPS, POP3S) to protect against interception.

PGP (Pretty Good Privacy): Encrypts and signs email content to ensure confidentiality and authenticity.

Digital Signatures: Verify message integrity and that the message originates from the stated sender.

2.8. Spam, Blacklists, Graylists, Spam Filtering

Spam: Unsolicited bulk email, often commercial or malicious.

Blacklists: Lists of known spammers or malicious IPs used to block or filter email.

Graylists: Temporarily reject messages from unfamiliar senders; legitimate servers retry, spam bots often do not, reducing spam.

Spam Filtering: Automated classification of messages as spam or not, possibly using heuristics, Bayesian filters, or reputation services.
